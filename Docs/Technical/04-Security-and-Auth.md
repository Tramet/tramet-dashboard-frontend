# 04 - Security & Authentication

Security in Tramet Dashboard is handled through a multi-layered approach involving **Next.js Middleware**, **JWT (JSON Web Tokens)**, and **Role-Based Access Control (RBAC)**.

---

## 🛡 Next.js Middleware
Path: [`middleware.ts`](../../middleware.ts)

The middleware intercepts every request before it reaches a page. Its primary responsibilities are:

1.  **Public vs. Protected Routes**: It identifies public routes (like `/login`) that don't require authentication.
2.  **Token Discovery**: It attempts to extract the JWT from the `auth-storage` cookie.
3.  **Role Verification**:
    - Decodes the JWT (base 64 decoding of the payload).
    - Checks the `role` field.
    - Redirects the user if they try to access a page they don't have permission for (e.g., a standard user trying to access `/admin`).
4.  **Graceful Redirection**: If a user is not logged in but tries to access a dashboard page, they are redirected to `/login`.

---

## 🔑 JWT Integration

### Token Storage
The session token is stored in a cookie named `auth-storage`. 
- **Type**: URL-encoded JSON string.
- **Origin**: Generated by Zustand's `persist` middleware. The raw cookie value looks like `{"state":{"token":"eyJhbG..."},"version":0}`.
- **Security**: It is used by both the middleware (server-side, decoded via `decodeURIComponent` and `JSON.parse`) and the `api-client` (client-side).

### Roles Supported
- **`TRAMET_ADMIN`**: Full access to system settings, sites, users, and departments.
- **`CUSTOMER_ADMIN`**: Access to managing their own users and viewing organizational data.
- **`USER`**: Standard operations, restricted to their assigned areas.

---

## 🚪 Authentication Flow

1.  **Login**: User submits credentials to `/auth/login`.
2.  **Token Issuance**: Backend returns a JWT.
3.  **Client-Side Save**: The `login` page saves the token in the `auth-storage` cookie.
4.  **Session Hook**: The `useAuth` hook monitors the token and provides helpers like `isAuthenticated` and `userRole`.
5.  **Logout**: The session is destroyed by clearing the `auth-storage` cookie and redirecting to `/login`.

---

## ⚡ Protection Patterns

### Route Protection
Paths are protected in the middleware using a prefix-based logic:
```typescript
if (path.startsWith("/admin") && role !== "TRAMET_ADMIN" && role !== "CUSTOMER_ADMIN") {
  return NextResponse.redirect(new URL("/dashboard", req.url));
}
```

### API Protection
The `api-client` automatically appends the token to every request. If the backend returns a `401 Unauthorized` (indicating the token expired), the `api-client` will force a logout by clearing cookies and redirecting the user.
